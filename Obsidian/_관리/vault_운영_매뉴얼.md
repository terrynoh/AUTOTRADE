## 1. vault 의 자리매김

vault 는 AUTOTRADE 의 *역사 + 일별 흐름 + 결정 근거* 영구 저장소.

| 영역 | 단일 진실 |
|---|---|
| 현재 사실 (매매 명세, 파라미터, 시각, 손절선, 코드 구조) | **CLAUDE.md** (작업폴더) |
| 시간축 (일별 작업/매매/사고/결정) | **vault** |
| 외부 사양 (KIS API, 환경 setup, 코딩 규칙) | vault (정적 참조) |

vault 는 CLAUDE.md 옆에서 *시간축* 과 *심화 영역* 을 담당. 두 영역의 책임이 *겹치지 않게* 운영하는 것이 W-16 의 핵심 의도.

## 3. 일일 마무리 루틴

작업 종료 시 채팅에서 다음 패턴 그대로 던짐:

```
오늘 vault 마무리 부탁.
- 거래일: Y / N
- 사고/빈틈: Y / N
- 결정: Y / N
- R-N 진행: <R-08 / R-09 / N/A>
- 오늘 한 일 요약: <한 줄~세 줄>
```

Claude 는 본 매뉴얼 §3 을 인지하고 자동으로 다음을 수행:

### 3.1 개발일지 갱신 (필수, 모든 작업일)

`개발일지/YYYY-MM-DD.md` 신규 또는 추가. 템플릿 = `개발일지/_템플릿.md`.

항목:
- 오늘 한 일 (W-N / M-N / 사고 대응 / 일반 작업)
- 발견 (빈틈, 의문, 검증 필요 영역)
- 메모 (생각, 결정 근거)
- 다음 할 일
- 이슈 (있으면 ISSUE-NNN 링크)

### 3.2 매매일지 작성 (거래일 = Y)

`매매일지/YYYY-MM-DD.md` 신규. 템플릿 = `매매일지/_템플릿.md` (R-08 영역).

핵심 항목:
- 시장 환경 (KOSPI / KOSDAQ / KOSPI200 선물)
- 입력 종목 (개수 제한 없음)
- 09:50 스크리닝 결과 (전부 통과 / 일부 탈락)
- Watcher 별 상태 추적 (8 상태)
- T1 / T2 / T3 발생 여부 + 시각 + ReservationSnapshot
- 청산 내역 (5 조건 중 무엇)
- N차 매매 연쇄 (몇 차까지)
- last-line defense 발화 여부 (정상 = 0건)
- 빈틈 발견 → ISSUE-NNN 등록

### 3.3 이슈 트래커 갱신 (사고/빈틈 = Y)

`이슈_트래커.md` 에 ISSUE-NNN 신규 등록 또는 상태 변경. 양식 = 트래커 상단 템플릿.

### 3.4 결정 문서 작성 (결정 = Y)

`결정/YYYY-MM-DD-주제.md` 신규.

항목:
- 결정 (한 줄)
- 근거 (사실 base)
- 대안 (검토했으나 채택 안 한 길)
- 영향 영역 (코드 / 명세 / 운영 / vault 어디에 영향)
- 후속 작업 (필요 시 W-N)

### 3.5 R-N 종결 문서 갱신 (R-N 종결 시)

`R/R-N_종결.md` 작성 또는 갱신. R-N 이 종결되는 *그 날* 즉시. 미루면 100% 누락.

### 3.6 비대칭 점검

활성 영역 신규/변경 내용이 CLAUDE.md 와 충돌하지 않는지 빠른 확인. 충돌 발견 시 *즉시 보고*, 자동 수정 금지.

## 4. 신규 Claude 채팅 활성화 시

신규 채팅 시작할 때 업로드 순서:

1. **CLAUDE.md** (작업폴더, 최신 버전) — anchor
2. **본 매뉴얼** (`_관리/vault_운영_매뉴얼.md`) — vault 운영 패턴

두 파일만 정독시키면 동일 패턴 자동 인계. CLAUDE.md §14 가 본 매뉴얼을 명시 참조하므로, 신규 채팅은 옵션 C 구조를 즉시 인식.

추가 자료가 필요한 경우 (예: 특정 R-N 상세, ISSUE 이력) 만 추가 업로드. 매번 vault 전체를 올릴 필요 없음.

## 5. 비대칭 사고 방지 가드 3 가지

vault 가 다시 stale 되는 것을 막기 위한 가드. R-08 이전 vault 가 4/7~4/10 사이 시간 정지된 사고의 직접 원인이 *동기화 책임 불분명* 이었음.

### 가드 1 — stub 영역 절대 갱신 금지

`00_대시보드.md`, `02_Phase_진행상황.md` 의 stub 부분에 현재 사실을 적는 순간 stale 시드 발생. 이 두 파일은 *링크 + history* 만.

### 가드 2 — R-N 종결 시 즉시 R/R-N_종결.md 작성

R-N 이 종결되면 *그 날* 종결 문서 작성. 다음 R-N 으로 넘어가면 디테일 휘발.

### 가드 3 — 월 1회 stale 점검 (또는 R-N 종결 직후)

다음 명령으로 점검:

```
vault 활성 영역과 CLAUDE.md 정합 점검 부탁.
```

Claude 는 매매일지/이슈/결정 문서가 CLAUDE.md 동결값과 충돌하지 않는지 grep 기반 점검.

## 6. 금지 사항

1. **stub 영역에 현재 사실 적기** — stale 시드
2. **`_archive/` 의 stale 파일을 정답지로 참조** — 사실 base 위반
3. **vault 작업 중 AUTOTRADE 작업폴더 또는 운영 서버 변경** — 영역 분리 위반
4. **R-N 종결 문서 작성 미루기** — 가드 2 위반
5. **`_archive/` 파일 삭제** — vault 원본 보존 원칙 (CLAUDE.md §15-16)
