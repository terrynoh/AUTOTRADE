- (나) EXITED 시 새 Watcher 인스턴스 생성 → 구조 변경 큼

### [R14-005] 청산 시 예수금 미복원 버그
- **발생일**: 2026-04-17
- **종결일**: 2026-04-17
- **R-N**: R-14
- **심각도**: 🔴 Critical
- **상태**: ✅ 해결완료

#### 현상
첫 테스트 거래(퍼스텍) 목표가 청산 후 예수금이 ~25M으로 표시 (50M으로 복원되지 않음).

#### 로그 분석
```
10:07:18  [동결] buy1  24,990,750원
10:07:18  [동결] buy2  24,987,280원
10:07:18  [확정] buy1  24,990,750원  사용→24,990,750 ✅
10:10:11  [해제] buy2  24,987,280원  buy2 취소
10:10:12  [복원] 0원  P&L -24,990,750원  ← 버그!
```
`restore(0원)` 호출됨 = `exit_value = 0`

#### 원인
`main.py:_on_exit_done`에서:
```python
exit_value = watcher.current_price * self.trader.position.total_qty if self.trader.position else 0
```
`self.trader.position`이 `None`이면 `exit_value = 0`이 됨.

**근본 원인**: `self.trader.position`이 왜 None인지는 미확정 (디버그 로그 추가함)

#### 해결
`watcher.total_buy_qty` 사용으로 변경:
```python
exit_value = watcher.current_price * watcher.total_buy_qty if watcher.total_buy_qty > 0 else 0
```
- `watcher.total_buy_qty`는 `on_buy_filled`에서 항상 갱신되어 신뢰성 높음
- `trader.py:simulate_fills`에 디버그 로그 추가 (근본 원인 추적용)

#### 관련
`src/main.py`, `src/core/trader.py`
