### [R08-009] R-08 재매수 명세-구현 갭
- **발생일**: 2026-04-10 (M-20)
- **R-N**: R-08 백로그
- **심각도**: 🟡 Medium
- **상태**: 미해결

#### 현상
명세 (수석님 확정 2026-04-10): 동일 종목 당일 재매수 허용. EXITED 후 같은 종목이 다시 신고가 갱신 + 1% 하락 시 재매수 가능.
구현: `Coordinator.on_realtime_price` (line 236) + `Watcher.on_tick` (line 693) 두 겹에서 EXITED Watcher 의 tick 차단 → 재트리거 구조적 불가능.

#### 원인
`on_tick` 진입 가드가 `not is_terminal` 에서 차단. `is_terminal` = DROPPED / EXITED / SKIPPED 모두 포함.

#### 영향
명세 ≠ 구현 갭. 첫 거래일 (4/13) 에 동일 종목이 EXITED 후 신고가 재갱신하면 *재매수 못 함*. 명세 위반 사고는 아니지만 의도 미구현.

#### 수정 방향
3 갈래 결정 필요:
- (가) `on_tick` 가드를 `state == DROPPED` 만으로 좁힘 → EXITED 도 다시 WATCHING 으로 reset 해야 함
- (나) EXITED 시 새 Watcher 인스턴스 생성 → 구조 변경 큼
- (다) 명세를 실 구현에 맞춰 수정 (재매수 불가 확정)

우선순위: 첫 거래일 검증 후 사례 발생 빈도 보고 결정.

### [R08-011] Pydantic validator 부재 (시각 필드)
- **발생일**: 2026-04-10 (W-15)
- **R-N**: R-08 백로그
- **심각도**: 🟢 Low
- **상태**: 미해결

#### 현상
`config/settings.py` 의 `entry_deadline`, `force_liquidate_time`, `repeat_start`, `repeat_end` 등 시각 필드가 `str` 타입으로 직접 할당. yaml 의 잘못된 형식 (`"25:00"`, `"abc"`) 입력 시 런타임 에러로만 발견.

#### 원인
Pydantic field validator 미구현. 타입 검증만 있고 *형식 검증* 없음.

#### 영향
yaml 손상 시 가동 후 첫 트리거 시점에 에러. 자본 위험 직접 0 이지만 *조용히 잘못된 시각이 통과* 할 수 있음 (예: `"10:5"` 가 `time.fromisoformat` 에서 `10:05` 로 해석되는 경계).

#### 수정 방향
`@field_validator` 추가, ISO 형식 (`HH:MM` 또는 `HH:MM:SS`) 강제. W-N 영역.
