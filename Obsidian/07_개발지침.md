## 8. 절대 금지사항

1. DRY_RUN 없이 실거래 가동 — 수석님 명시 결정 전까지 LIVE 모드 자동 전환 금지
2. 매매 명세 임의 변경 — yaml 동결값. 변경 시 새 R-N redesign 필수
3. Coordinator / Watcher / Trader 상태 전이 로직 임의 변경 — 검증 없이 금지
4. 운영 환경 직접 코드 수정 — 로컬 → 검증 → 배포 순서 엄수
5. 추측 기반 작업 — 사실 base 우선
6. 화이트리스트 외 파일 수정 — W-N 명령서 화이트리스트 엄수
7. commit 자동 실행 — 수석님 결정
8. 테스트 없는 배포 — 로컬 검증 → 운영 배포 → 운영 검증 3단계
9. 중간 멈춤 없는 자동 진행 — 멈춤 조건 충족 시 즉시 보고

## 9. Claude §5.6 원칙 1 위반 패턴 (학습)

R-08 진행 중 반복 발생한 패턴:

1. 메모리/이전 대화 값을 사실 base 로 단정 → grep 재확인 안 함
2. 명세 사실 base 무시 → 코드는 grep, 명세는 머릿속 가정으로 처리
3. 무의식적 가정 → "재매수 = 사고" 같은 도메인 가정을 명세 근거 없이 적용
4. 컨텍스트 길어질수록 사실 base 확인 게을리짐
5. 답변 모호 시 좁게 해석 → 양쪽 가능한 경우에도 한쪽으로 단정

**대응**: 모든 결정 전 "이 결정의 사실 base 가 무엇인가" 자문.
