## 근거 (사실 base)

### 즉발 원인
- Quick Tunnel 의 URL 불안정성 (재기동마다 변경, 과거 URL 즉시 DNS 만료)
- `send_dashboard_url.py` 의 `autotrade.err` 누적 로그 오염 — W-18 로 임시 해결했으나 근본은 Named Tunnel 전환
- ISSUE-028 (Named Tunnel 전환) 이 α-1b 영역에 계류 중이었음

### 확장 욕구
- 수석님이 정식 서비스 스펙 (2 tenant, API 호출 고려) 을 "한 번에 제대로 된 방식으로" 구축 의지 표명
- Phase α-1 (멀티 사용자 UI) + α-1b (Named Tunnel) + α-2 (Account Executor) 를 통합 프로젝트로 묶어 R-10 으로 명명

### Oracle Cloud Always Free 사실 base (web_search 확정)
- **Ampere A1.Flex**: 4 OCPU + 24 GB RAM (단일 VM 또는 최대 4 분할), 영구 무료
- **API 콜 수 제한 없음**, **사용자 수 제한 없음**, 리퀘스트 수 제한 없음
- 블록 스토리지 200 GB, 아웃바운드 월 10 TB
- 유휴 회수 조건: CPU 95 퍼센타일 20% 미만 7 일 지속 (AUTOTRADE 장중 돌면 기준 훨씬 초과)
- Out of Capacity 회피: Pay As You Go 업그레이드 (카드 $100 일시 인증, 실과금 0)
- 출처: `docs.oracle.com/en-us/iaas/Content/FreeTier/`, `oracle.com/cloud/free/`, 교차 검증 소스 다수

### 2 tenant 부하 평가
- WebSocket 2~4 개 (계정당 1~2), KIS REST 수백~수천/일, asyncio loop 1~2, FastAPI 동시 접속 2~4, RAM 1~3 GB, CPU 평균 5~15%
- → A1.Flex (4 OCPU / 24 GB) 에서 **매우 여유 있음** (RAM 사용률 ~12%, CPU 여유 85%+)
- SQLite WAL 모드는 2 tenant 동시 쓰기 감당 가능. PostgreSQL 마이그레이션 불필요

## 대안 (검토했으나 채택 안 함)

### (가) Quick Tunnel 유지 + W-18 영구 사용
- 장점: 비용 0, 즉시 적용
- 단점: URL 매일 변경, 북마크 불가, 텔레그램 의존, 2 tenant 에서 운영 불편
- **탈락**: 근본 해결 아님

### (나) 기존 도메인 이전 (비 Cloudflare Registrar)
- 장점: 기존 자산 활용
- 단점: 네임서버 전파 24~48 시간, 월요일 전 완료 불투명
- **탈락**: 전환 비용 > 신규 구매 비용 ($10/년)

### (다) .kr 도메인
- 장점: 한국 서비스 정체성
- 단점: Cloudflare Registrar 미지원, 가비아/후이즈 에서 별도 구매 + DNS 이전
- **탈락**: 설정 복잡도 증가

### (라) E2.1.Micro 유지 (현 인프라)
- 장점: 이관 비용 0, ARM 호환성 검증 불필요
- 단점: 1 vCPU / 956 Mi RAM 으로 2 tenant + 확장 기능 부담, 향후 재이관 필요
- **탈락**: "한 번에 제대로" 원칙 위배

### (마) PostgreSQL 마이그레이션 포함
- 장점: 다중 사용자 동시 쓰기 안정성, audit_log 성능
- 단점: 2 tenant 규모는 SQLite WAL 로 충분. 마이그레이션 리스크 + 시간 증가
- **탈락**: 과잉 설계. R-10 범위 제외, R-11+ 에서 재평가

### (바) AWS / GCP 유료
- 장점: 상용 SLA
- 단점: 월 $30~100 비용, 정식 서비스 수익 없는 단계에서 과투자
- **탈락**: 현 단계 경제성 없음
