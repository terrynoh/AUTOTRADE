## 배경

**ISSUE-LIVE-02**: Day 1 (4/17) 스크리닝에서 프로그램 순매수 비중이 0 또는 비정상 반환, 종목 대거 탈락.

### 근본 원인 (Day 2 진단 결과)

KIS `program-trade-by-stock` 의 `FID_COND_MRKT_DIV_CODE="UN"` (통합) 은 **KRX + NXT 합산 반환**. 단 NXT 미상장 종목은 UN 조회 시 **곧바로 빈 배열 반환** (fallback 없음).

### 실측 매트릭스 (2026-04-20 14:42, 5종목 × 3코드)

| 종목 | J | NX | UN | 분류 |
|------|---|-----|-----|------|
| 주성엔지 (036930) | len=30 | len=0 | **len=0** | NXT 미상장 |
| 우리기술 (032820) | len=30 | len=0 | **len=0** | NXT 미상장 |
| 에코프로머티 (450080) | len=30 | len=0 | **len=0** | NXT 미상장 |
| 로보티즈 (108490) | len=30 | len=30 | len=30 | NXT 이중상장 |
| 한화비전 (489790) | len=30 | len=30 | len=30 | NXT 이중상장 |

### UN = J + NX 산술 증명

로보티즈 (NXT 이중상장):
- J (KRX): 25.7B
- NX (NXT): 7.0B
- UN (통합): 32.7B ← **J + NX 완전 일치**

→ UN 은 단순 합산, 이중상장 종목은 UN 으로 메내지만 미상장은 빈 배열

## 방안 비교

### 방안 1 (탈락) — J 단독 사용 (전환)

- 모든 종목에 대해 UN 대신 J 사용
- 장점: 단순, 1회 호출로 해결
- 단점: **NXT 이중상장 종목의 NXT 분 프로그램매매 누락** (로보티즈 NXT 7.0B 날라감)
- NXT 비중이 높은 종목 (갈수록 증가 추세) 에서 매매 판단 왔곡 가능성
- 탈락 이유: 리그레션 발생 (기존 UN 정상 동작 종목들의 분석이 있하 바뀌어버림)

### 방안 2 (채택) — UN→J fallback

- 1차 UN 호출 후 빈 배열 시 “FID_COND_MRKT_DIV_CODE="J"” 재호출
- **NXT 이중상장 종목**: UN 정상 반환 → fallback 미발동 → 기존 동작 동일
- **NXT 미상장 종목**: UN 빈 배열 → J fallback → KRX 데이터 정상 사용
- 장점: **리그레션 0**. NXT 이중상장 종목은 기존 동작 유지, 미상장만 교정
- 단점: 날마다 1초마다 NXT 미상장 종목은 REST 2회 호출 (rate limit 부담 추가)
  - R-13 `_ratio_updater` 1초 폴링 중 NXT 미상장 종목 비율 높으면 rate limit 지연 누적 지켠 필요

## 구현 상세

```python
# src/kis_api/kis.py::get_program_trade

# 1차: UN (통합) — 기존 동작
params = {"FID_COND_MRKT_DIV_CODE": "UN", "FID_INPUT_ISCD": code}
data = await self._get(EP_PROGRAM_TRADE_BY_STOCK, TR_PROGRAM_TRADE_BY_STOCK, params)
output = data.get("output", [])
source = "UN"

# M-LIVE-02-FIX: UN 빈 배열 시 J fallback (NXT 미상장 종목 대응)
if not (isinstance(output, list) and output):
    logger.info(
        f"[PROG-FALLBACK-J] {code} UN=EMPTY → J(KRX) 재조회 "
        f"rt_cd={data.get('rt_cd')!r}"
    )
    params["FID_COND_MRKT_DIV_CODE"] = "J"
    data = await self._get(EP_PROGRAM_TRADE_BY_STOCK, TR_PROGRAM_TRADE_BY_STOCK, params)
    output = data.get("output", [])
    source = "J-FALLBACK"

# UN + J 모두 빈 배열 → 진짜 이상 (PROG-ANOMALY-EMPTY)
if not (isinstance(output, list) and output):
    logger.warning(f"[PROG-ANOMALY-EMPTY] {code} UN+J 모두 empty ...")
    return {"program_net_buy": 0, "buy_amount": 0, "sell_amount": 0}
```

**반환 스키마 불변** → 호출자 3곳 (screener, 관련 호출, `_ratio_updater`) 수정 불필요.
