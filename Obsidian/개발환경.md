## 환경 개요

서버 2개 (로컬 + 운영). 별도 stage 서버 없음.
**stage 역할은 `feature/*` 브랜치**가 대신함.

```
로컬 PC (Windows 11)          git origin (GitHub)          운영 서버 (Oracle Cloud)
─────────────────────         ──────────────────           ────────────────────────
feature/*             →push→  feature/*                    /home/ubuntu/AUTOTRADE/
                                   ↓ (배포 성공 후 머지)        ↑ git pull origin main
main                  →push→  main  ───────────────────────────┘
```

---

## 브랜치 구조

| 브랜치 | 역할 |
|---|---|
| `main` | **운영 서버에 배포된 코드** (단일 진실) |
| `feature/*` | 작업 브랜치 (stage 역할) |

**핵심 규칙**: main 머지 = 배포 성공 도장. **배포 전 main 머지 금지.**

---

## 배포 흐름

```
1. feature 브랜치에서 작업 + commit + push
2. 운영 서버: git pull origin main
3. py_compile 검증
4. systemctl restart autotrade
5. 운영 정상 확인
6. main fast-forward merge + push  ← 반드시 5번 이후
```
