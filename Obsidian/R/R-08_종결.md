- 1차 진입 빈도 ↑
- 1차를 *얕게* (덜 떨어진 시점에 매수)

## 두 번째 매매 체인 (W-11d/e 핵심)

### T1 (첫 종목 ENTERED)
`Coordinator._active_code` 가 첫 종목 코드로 설정.

### T2 (첫 종목 buy2 체결)
1. `Watcher.on_buy_filled` 의 buy2 분기에서 `_t2_callback` 호출
2. `Coordinator._on_t2` 진입 (동기)
3. `_is_in_entry_window` (10:00~10:55) 체크
4. YES 후보 수집 (`is_yes` = state == READY)
5. **tiebreaker**: KOSPI 눌림폭 ≥ 3.8% / KOSDAQ ≥ 5.6% 필터 + 눌림폭 최대
6. 통과 종목을 `_reserved_snapshot` (ReservationSnapshot dataclass) 에 저장

### T3 (첫 종목 EXITED)
1. `main.py._on_exit_done` 의 6.5 단계: `_active_code = None` + `await coordinator.handle_t3(now_kst())`
2. `Coordinator.handle_t3` (async):
   - 진입 윈도우 체크 (벗어나면 예약 폐기 + return)
   - 예약 존재 → `_verify_reservation_at_t3` (5 항목 엄격 재확인)
     1. 종목이 watchers 에 존재
     2. is_yes (state == READY)
     3. not is_terminal
     4. target_buy1_price 가 T2 시점 값과 동일 (재계산되지 않음)
     5. tiebreaker 조건 여전히 충족
   - 통과 → `_execute_buy` (예약 진입)
   - 실패 → 예약 폐기 + fallthrough 재판정 (yes_watchers 재수집 + tiebreaker)
   - 재판정 0 개 → 즉시 포기

### N차 매매 연쇄
두 번째 매매에서도 T2/T3 가 발생하면 세 번째, 네 번째 매매가 이어짐. 진입 윈도우 (10:00~10:55) 가 끝나면 자연스럽게 멈춤. `daily_loss_limit_pct = 3.0%` 가 누적 손실 안전망.

### Trader.place_buy_orders last-line defense (W-11e)
```python
_now = now_kst()
_repeat_end_time = dtime.fromisoformat(self.params.multi_trade.repeat_end)  # "11:00"
if _now.time() >= _repeat_end_time:
    logger.error(f"CRITICAL: 매매 철학 시한 위반 시도 ... 발주 거부")
    return
```

11:00 이후 어떤 경로로도 발주 거부. 정상 동작 시 영원히 안 찍혀야 함.
