### [ISSUE-035] Dashboard set-targets aiohttp TimerContext RuntimeError
- **발생일:** 2026-04-08
- **Phase:** Phase α-1a
- **심각도:** 🔴 Critical (Dashboard 수동 종목 입력 100% 실패)
- **상태:** 미해결 — 원인 확정, 수정 α-1b/α-2

#### 현상
Dashboard 수동 종목 입력 후 "종목 설정" 클릭 시 100% 실패. 장 중/휴장 무관.
```
RuntimeError: Timeout context manager should be used inside a task
```

#### 원인
AutoTrader event loop에서 생성한 `aiohttp.ClientSession`을 FastAPI endpoint(uvicorn)의 다른 Task 컨텍스트에서 직접 await. aiohttp timeout `TimerContext.__enter__`가 `asyncio.current_task() == None` 감지 → RuntimeError.

#### 호출 경로
```
app.py api_set_targets()
  → state.autotrader.api.get_current_price()
    → KISAPI._request()
      → aiohttp.ClientSession.get()
        → TimerContext.__enter__() → RuntimeError
```

#### 영향
- ❌ Dashboard 수동 종목 입력 100% 실패
- ✅ 자동 스크리닝 (AutoTrader 내부 호출) 정상
- ✅ 매매 로직 (KIS WebSocket + REST) 정상
- ✅ 텔레그램 `/target` 명령 정상 (AutoTrader 내부 호출)

#### 운영 우회
텔레그램 `/target 코드,코드` 명령 사용. Dashboard 종목 입력 기능 사용 안 함.

#### 수정 방향
(B) 선호 — FastAPI endpoint가 asyncio queue 패턴으로 AutoTrader event loop에 위임.
수정 시기: α-1b 또는 α-2 (dashboard 구조 개선 시)

#### 삭제 근거
1. 백테스트 3종목 모두 TREND_BREAK가 수익 저해 방향으로 작동
2. 하드손절 + 25분 타임아웃 + 선물급락이 하방 안전장치로 충분
3. 1분봉 higher low 판단의 신뢰도가 낮아 오작동 확률 높음
4. 최악 추가 손실은 약 1.5~3%p 수준이며 shakeout 방지 이점이 더 큼

#### 변경 내용
- 6중 청산 → **5중 청산** (하드손절/25분타임아웃/목표가/선물급락/강제청산)

### [ISSUE-005] Cloudflare Tunnel 502 Bad Gateway — 대시보드 서버 미기동
- **발생일:** 2026-04-06
- **Phase:** Phase 3
- **심각도:** 🔴 Critical
- **상태:** 해결완료

#### 현상
`python -m src.main` 실행 후 Cloudflare Tunnel URL 접속 시 `502 Bad Gateway` 반복 발생.
여러 차례 재시작, 포트 확인, URL 재생성을 반복했으나 동일 증상 지속.

#### 원인
`main.py`가 `CloudflareTunnel(port=8503)`으로 **Tunnel만 시작**하고,
실제 HTTP 요청을 받을 **대시보드 서버(uvicorn)를 기동하지 않았음.**

```
CloudflareTunnel → localhost:8503 → (아무것도 없음) → 502
```

대시보드(`app.py`)는 원래 별도 프로세스(`python -m src.dashboard.app`)로 실행하는 설계였으나,
`main.py`에서 Tunnel을 시작하면서 대시보드 서버를 함께 시작하는 코드가 누락됨.

#### 왜 반복됐는가
- 포트 불일치(8501→8503), 이전 URL 재사용, 셸 세션 종료 등 **표면적 원인**에만 집중
- "localhost:8503에 서버가 리슨 중인가?" 라는 **근본 체크를 하지 않음**
- `netstat -an | grep 8503` 또는 `curl http://localhost:8503` 한 번이면 즉시 발견 가능했음

#### 해결
`src/main.py`에 대시보드 서버를 데몬 스레드로 시작하는 `_start_dashboard_server()` 추가:

```python
def _start_dashboard_server(self) -> None:
    import uvicorn
    from src.dashboard.app import app as dashboard_app
    port = self.settings.dashboard_port
    def _run():
        uvicorn.run(dashboard_app, host="0.0.0.0", port=port, log_level="warning")
    t = threading.Thread(target=_run, daemon=True, name="dashboard")
    t.start()
```

실행 흐름 변경:
```
AutoTrader.run()
  → _start_dashboard_server()  ← uvicorn 포트 8503 리슨
  → _tunnel.start()            ← cloudflared → localhost:8503 → uvicorn ✅
```

#### 재발 방지 → [[07_개발지침]] 규칙 8 추가

token_param = request.query_params.get("token", "")

**1) 대시보드와 main.py가 독립 프로세스 — 상태 공유 안 됨**

2. `fmtWon(0)` → `'0원'` 반환하도록 수정
3. P&L: `daily_pnl > 0` 일 때만 `+` prefix (0일 때 제외)

- 백엔드 수정 시 대시보드 HTML/JS와의 연동 반드시 확인
- 숫자 포맷 함수 수정 시 `0`, `null`, `undefined`, 음수, 양수 5가지 케이스 검증
