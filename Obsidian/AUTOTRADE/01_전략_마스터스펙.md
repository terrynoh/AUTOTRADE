## 3. 스크리닝 — 수동 입력 + 자동 검증

> **KIS API에 프로그램 순매수 순위 엔드포인트 없음** → 1단계 수동 입력 방식 채택

### 입력 채널
| 채널 | 사용법 |
|------|--------|
| 대시보드 UI | 종목코드 텍스트 입력 → "종목 설정" → "수동 스크리닝" |
| 텔레그램 | `/target 006400,247540,020150` · `/clear` · `/target` (조회) |

### 자동 검증 필터 (순서대로)

```
프로그램순매수비중(%) = (프로그램 순매수금액 / 누적 거래대금) × 100
```

| 순서 | 조건 | yaml 키 |
|------|------|---------|
| 1 | 상승률 > 0% | — |
| 2 | 거래대금 ≥ 500억 원 | `screening.volume_min` |
| 3 | 상한가 미도달 (등락률 < 20%) | `infra.upper_limit_check_pct` |
| 4 | 프로그램순매수비중 ≥ 5% | `screening.program_net_buy_ratio_min` |
| 5 | 상승률 최고 1종목 | `screening.top_n_gainers` |

> ⚠️ 조건 미충족 = 당일 매매 안 함
> 수동 종목 미설정 시 → 자동 스크리닝(거래량순위 API) 폴백

## 4. 매수 진입 (09:55~ / 진입 마감 11:00)

### 고가 확정 트리거
신고가 달성 후 **고가에서 1% 하락** 시 → 고가 확정, 매수 주문 2건 진입

### 분할매수 (지정가)

| 시장 | 1차 (예수금 50%) | 2차 (예수금 50%) |
|------|----------------|----------------|
| KOSPI | 고가 대비 -2.5% | 고가 대비 -3.5% |
| KOSDAQ | 고가 대비 -3.75% | 고가 대비 -5.25% |

### 고가 갱신 시 재조정
1. 현재가 > 기존 고가 → 미체결 매수 전량 취소
2. 새 고가에서 1% 하락까지 대기
3. 새 고가 기준으로 2건 재주문

## 10. 주요 제약사항

- **pykrx 장중 호출 금지** — 느리고 타임아웃/IP차단 위험. 장중은 전부 KIS API
- **KIS Rate Limit** — 실거래 초당 20건 / 모의투자 초당 2건 (asyncio.Semaphore)
- **전략 수치 하드코딩 금지** — 전부 `config/strategy_params.yaml`에서 로드
- **프로그램매매 ≠ 기관 순매수** — 차익/비차익 프로그램 주문만 포함
